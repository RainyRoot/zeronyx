ZeroNyx Documentation

Welcome to ZeroNyx — the desktop-based all-in-one pentesting suite. This documentation covers installation, all core features, and the Plugin SDK.

Introduction

ZeroNyx is built on three principles:

  • Orchestrator, not bundler — ZeroNyx launches external tools as subprocesses and parses their output. Your existing tool installations just work.
  • Project-based — Every engagement is a separate project with its own SQLite database. Data stays local and organized.
  • Free must be good — The Community edition is not a demo. It includes all scanning tools, findings management, the proxy, and credential store.

Installation

Download

Download the latest release for your platform from GitHub Releases:

  • Linux — .AppImage or .deb
  • macOS — .dmg
  • Windows — .exe installer

External Tools

ZeroNyx requires the tools to be installed separately on your system. Install them via your package manager:

# Kali / Debian / Ubuntu
sudo apt install nmap nuclei nikto gobuster hydra sqlmap exploitdb

# Arch
sudo pacman -S nmap hydra sqlmap

# macOS (Homebrew)
brew install nmap hydra sqlmap
ZeroNyx will auto-detect tools on first launch and show their status in Settings → Tool Health.

Quick Start

  1. Launch ZeroNyx and create a new Project (name + scope).
  2. Add your target(s) in the Targets tab (IP, domain, CIDR, or URL).
  3. Click New Scan, choose a tool, configure parameters, and run.
  4. View real-time output in the terminal pane, then review parsed results.
  5. Findings are automatically normalized and stored under Findings.

Projects

Each project is a separate engagement. ZeroNyx stores all project data in a dedicated SQLite file at ~/.zeronyx/projects/{name}.db. Projects can be archived when complete.

Scanning

ZeroNyx supports the following tools via the Unified Scan Interface:

  • nmap — Port scanning, OS detection, service fingerprinting
  • nuclei — Template-based vulnerability scanning
  • nikto — Web server misconfiguration scanning
  • gobuster / ffuf — Directory and subdomain enumeration
  • hydra — Credential brute-forcing
  • sqlmap — SQL injection detection and exploitation
  • metasploit — Exploit framework integration
  • searchsploit — ExploitDB search
  • shodan / censys — OSINT and passive recon

All scan output is streamed live via WebSocket and stored in the database for later review.

HTTP(S) Proxy

ZeroNyx includes a full mitmproxy integration. Configure your browser or tool to use 127.0.0.1:8080 as an HTTP proxy. The Proxy page shows all intercepted requests with:

  • Full request/response headers and body
  • Search and filter by host, method, status code
  • Replay any request
  • Modify and forward requests (Pro)

AI Analysis

Connect an AI provider in AI Analysis → Settings:

  • Ollama (default) — Fully local, no API key, complete privacy
  • OpenAI — GPT-4 for complex analysis
  • Anthropic — Claude for detailed technical reports

After any scan, click Analyse with AI to get:

  • Risk-prioritized finding summary
  • False-positive identification
  • Concrete next-step recommendations
  • Full engagement report generation
AI Analysis requires ZeroNyx Pro.

Chain Automation

Chains define multi-step workflows that run automatically. Built-in chains:

  • Quick Recon — Nmap → SearchSploit → Shodan
  • Full Web Audit — Nmap → Nikto → Gobuster → Nuclei
  • Network Sweep — Nmap sweep → per-host port scans
  • Credential Attack — Hydra against discovered services

Chains can be triggered manually, on scan completion, or on a schedule. Custom chains can be built in the Chains UI.

Chain Automation requires ZeroNyx Pro.

Plugins

Plugins are .zeronyx-plugin archives (renamed zip files) containing:

  • A manifest.json defining metadata, permissions, hooks, and UI slots
  • An optional Python backend module
  • An optional pre-compiled JavaScript frontend bundle

Install plugins via Plugins → Install (drag & drop) or from the Marketplace tab with one click.

REST API

The backend exposes a full REST API at http://127.0.0.1:8742. In development mode (ENV=dev), interactive API docs are available at /docs.

GET  /api/projects
POST /api/projects
GET  /api/targets
POST /api/scans/run
GET  /api/findings
GET  /api/license/status
POST /api/license/activate
GET  /api/marketplace

Plugin SDK

See the PLUGIN_SDK.md for the full developer reference. Quick example:

from zeronyx.sdk import ZeroNyxPlugin, PluginContext

class MyPlugin(ZeroNyxPlugin):
    async def on_scan_complete(self, ctx: PluginContext) -> None:
        scan = ctx.scan
        await ctx.api.create_finding(
            title=f"Custom check on {scan.target}",
            severity="info",
        )

License System

ZeroNyx uses offline-capable RSA-signed JWT license keys. After purchase, you receive a key by e-mail. Activate it in Settings → License.

License keys are verified against an embedded RSA public key — no internet connection required for activation or verification.
